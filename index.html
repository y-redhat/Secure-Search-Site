Security & Anonymity

What is protected?

  • Destination URLs & search queries (AES-GCM encryption)
  • DNS leaks prevented (socks5h://)
  • Internal network / metadata protected (SSRF filtering)
  • Session token authentication (SHA-3 password)
  • Redirect chain validation

What is not hidden?

  • Domain of the proxy (trycloudflare.com)
  • Traffic volume & timing
  • Cloudflare / Google metadata (IP, timestamps)
  • Tor exit node may see unencrypted HTTP
  • Fingerprinting by DPI

Can it bypass national firewalls (e.g., GFW)?

  • Can bypass: Basic tracking, content filtering based on keywords, ISP surveillance (traffic content hidden).
  • Cannot bypass: Deep Packet Inspection (DPI) that recognizes Tor/Cloudflare patterns; IP/domain blacklisting of known proxy services; active probing (if the firewall blocks Tor bridges).

🔹 For higher resilience, consider using WebTunnel bridges, custom domains, or VLESS+REALITY.

セキュリティと匿名性

保護されるもの

  • アクセス先URL・検索クエリ（AES-GCM暗号化）
  • DNS漏洩防止（socks5h://）
  • 内部ネットワーク・メタデータ保護（SSRFフィルタ）
  • セッショントークン認証（SHA-3パスワード）
  • リダイレクトチェーン検証

隠せないもの

  • プロキシのドメイン（trycloudflare.com）
  • 通信量・タイミング
  • Cloudflare/Googleのメタデータ（IP, 時刻）
  • Tor出口ノードでの平文HTTP
  • DPIによるフィンガープリント

国家レベルのファイアウォールは突破できるか？

  • 突破可能：キーワードフィルタリング、ISP監視（コンテンツは隠蔽）
  • 突破困難：DPI（Tor/Cloudflareパターン検出）、既知プロキシのIP/ドメインブラックリスト、能動的プロービング（Torブリッジがブロックされた場合）

🔹 より高い耐性を得るには、WebTunnelブリッジ、カスタムドメイン、VLESS+REALITYの導入を検討してください。

  • !ps aux | grep "python.*app.py" – 检查 Flask
  • !cat /content/tunnel.log – 获取 Cloudflare Tunnel URL

安全与匿名性

受保护的内容

  • 目标URL和搜索查询（AES-GCM加密）
  • 防止DNS泄露（socks5h://）
  • 内部网络/元数据保护（SSRF过滤）
  • 会话令牌认证（SHA-3密码）
  • 重定向链验证

无法隐藏的内容

  • 代理域名（trycloudflare.com）
  • 流量大小和时间
  • Cloudflare/Google的元数据（IP、时间戳）
  • Tor出口节点可能看到未加密的HTTP
  • DPI指纹识别

能否绕过国家级防火墙（例如GFW）？

  • 可以绕过：基本跟踪、基于关键词的内容过滤、ISP监控（内容被隐藏）
  • 难以绕过：深度包检测（识别Tor/Cloudflare模式）、已知代理的IP/域名黑名单、主动探测（如果Tor桥被封锁）

🔹 要获得更高的抗性，请考虑使用 WebTunnel 桥接、自定义域名或 VLESS+REALITY。
