Change password hash

              
This system uses SHA-3-256 hashing. Replace the default hash with your own.

              

                # Generate new hash for your password (e.g., "mysecret")

                echo -n "mysecret" | sha3sum -a 256
              

              
Then update EXPECTED_HASH in the code.

Kill all processes (stop)

              
To stop everything, run this kill script in a new cell:

 Security & Anonymity

          

            

              
 What is protected?

              
    
                
  • Destination URLs & search queries (AES-GCM encryption)
    • 
                
  • DNS leaks prevented (socks5h://)
    • 
                
  • Internal network / metadata protected (SSRF filtering)
    • 
                
  • Session token authentication (SHA-3 password)
    • 
                
  • Redirect chain validation
    • 
              

            

            

              
 What is not hidden?

              
    
                
  • Domain of the proxy (trycloudflare.com)
    • 
                
  • Traffic volume & timing
    • 
                
  • Cloudflare / Google metadata (IP, timestamps)
    • 
                
  • Tor exit node may see unencrypted HTTP
    • 
                
  • Fingerprinting by DPI
    • 
              

            

          

          

            
 Can it bypass national firewalls (e.g., GFW)?

            
    
              
  • Can bypass: Basic tracking, content filtering based on keywords, ISP surveillance (traffic content hidden).
    • 
              
  • Cannot bypass: Deep Packet Inspection (DPI) that recognizes Tor/Cloudflare patterns; IP/domain blacklisting of known proxy services; active probing (if the firewall blocks Tor bridges).
    • 
            

            

              🔹 For higher resilience, consider using WebTunnel bridges, custom domains, or VLESS+REALITY.
